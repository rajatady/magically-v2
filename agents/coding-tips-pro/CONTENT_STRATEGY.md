# Coding Tips Pro — Content Strategy

## The Goal

**1,000,000 followers by September 19, 2026.** Reels are the primary growth engine. Carousels serve retention and saves.

## Growth Playbook

### Algorithm Priorities (ranked by actual impact on follower growth)
1. **Shares (DM sends)** — THE growth metric. One share = your content enters a new audience cluster. Shares compound via network effects. Everything else is linear. Design every reel so someone sends it to a group chat.
2. **Watch time + Replays** — Instagram pushes reels that get watched to the end and rewatched. A 12-second reel watched twice beats a 60-second reel watched once.
3. **Saves** — Signals quality to the algorithm. Builds retention with existing followers. Saves are the #1 metric for CAROUSELS, not reels.
4. **Organic comments (real discussion)** — Genuine debate/reactions in comments signal relevance. Prompted comments ("type X to get Y") are detected as engagement bait and deprioritized.
5. **Keywords in captions > hashtags** — Instagram search is semantic now. Write natural searchable captions.

### How Content Spreads
- Content spreads WITHIN peer clusters, not across demographics. An 18-year-old shares with other 18-year-olds.
- When Instagram sees a reel getting shares within a tight demographic cluster, it pushes to MORE people in that exact cluster.
- That's how 100 views becomes 100K views — not by reaching everyone, but by saturating one peer group which triggers distribution to adjacent identical peer groups.

### The Group Chat Test (MANDATORY for reels)
Before publishing any reel, answer: "Would an 18-year-old CS student send this to their developer group chat?" If you can't write the group chat message they'd send with the reel, the reel will not spread.

### Posting Cadence
- Carousels: 1-2 per day, alternating templates
- Reels: quality over quantity. One great reel > five mediocre ones.
- Never dump multiple posts at once — spacing lets each one get algorithmic attention

## Content Pillars

### 1. A/B Code Quiz (PRIMARY — highest engagement, CAROUSELS)
"Which snippet is better?" format. Two code options, swipe for the answer.
- Why it works: Interactive, creates curiosity gap, forces opinion, drives saves
- Previous data: 157 saves, 223 likes, 271 profile visits (boosted)
- Save-to-like ratio: 70-133%

### 2. Cheat Sheet / Reference Card (CAROUSELS)
"The complete X cheat sheet" — saveable reference for a concept.
- Why it works: Utility content that people bookmark
- Examples: Promise methods decoded, Array method map, CSS Grid cheatsheet

### 3. "You're Doing It Wrong" Pattern (CAROUSELS)
Show the common way, then reveal the modern/better way.
- Why it works: Ego challenge + learning = high engagement
- Examples: structuredClone vs JSON trick, using keyword vs try/finally

### 4. New Feature Spotlight (CAROUSELS)
Highlight features from ES2025, TypeScript 5.x, React 19, Next.js 15, etc.
- Why it works: Developers want to stay current, fear missing out
- Examples: using keyword, Promise.withResolvers, Iterator helpers

### 5. 10x Developer Reels (REELS — PRIMARY for growth)
Short, punchy reels that make developers massively more productive. Focus areas:
- **Claude Code hidden tips & tricks** — CLI shortcuts, slash commands, MCP servers, hooks, permission modes, headless automation, session management, worktrees, agent orchestration
- **AI-assisted coding workflows** — how to use AI tools to ship 10x faster
- **The code behind everything** — connect code to real-world impact (stories, not tutorials)
- **Developer productivity hacks** — terminal tricks, git workflows, IDE shortcuts that save hours

Reel rules:
- Under 18 seconds, hook in 2 seconds
- NOT tutorials — these are "holy shit I didn't know that" moments
- Center text, fill the frame, cinematic pacing
- Language can repeat for reels (not restricted like carousels)

## Content Rules

1. **Every post starts with an A/B question or a bold claim** — never start with "Today we'll learn..."
2. **Code must be real, runnable, and correct** — verify before posting
3. **Slide 1 is the hook** — must work as a standalone thumbnail in the grid
4. **Last slide has a teaser** for the NEXT post — creates follow incentive
5. **Caption includes a question** — drives comments
6. **Save-worthy content** — every carousel should have at least one "cheat sheet" or "reference" slide
7. **Max 6 slides** — attention drops after 5. Sweet spot is 4-5.
8. **Code font size minimum 17px** — must be readable on mobile

## Posting Cadence

- **Target**: 1 carousel per day
- **Best times**: Mornings IST (8-10am) or evenings (6-8pm)
- **Never**: Don't post 3+ in a day — dilutes per-post performance
